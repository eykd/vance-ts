## Entities (Better-Auth Managed)

Better-auth owns and manages these tables. They are generated by the better-auth CLI and applied as D1 migrations.

### `user` Table

Stores registered user accounts. The `name` field stores the user's display name (derived from the email prefix on registration if not provided).

| Column          | Type    | Constraints             | Notes                                                    |
| --------------- | ------- | ----------------------- | -------------------------------------------------------- |
| `id`            | TEXT    | PRIMARY KEY             | UUID generated by better-auth                            |
| `name`          | TEXT    | NOT NULL                | Display name                                             |
| `email`         | TEXT    | NOT NULL, UNIQUE        | Case-sensitive storage; better-auth normalises on lookup |
| `emailVerified` | BOOLEAN | NOT NULL, DEFAULT FALSE | Always false in this iteration (FR-013)                  |
| `image`         | TEXT    | NULL                    | Profile picture URL (not used in this iteration)         |
| `createdAt`     | TEXT    | NOT NULL                | ISO 8601 UTC timestamp                                   |
| `updatedAt`     | TEXT    | NOT NULL                | ISO 8601 UTC timestamp                                   |

### `session` Table

Stores active user sessions. Better-auth validates `expiresAt` on every request. Sessions are deleted on logout.

| Column      | Type | Constraints                            | Notes                                  |
| ----------- | ---- | -------------------------------------- | -------------------------------------- |
| `id`        | TEXT | PRIMARY KEY                            | UUID generated by better-auth          |
| `userId`    | TEXT | NOT NULL, FK → user(id) CASCADE DELETE | Links session to user                  |
| `token`     | TEXT | NOT NULL, UNIQUE                       | The session token stored in the cookie |
| `expiresAt` | TEXT | NOT NULL                               | ISO 8601 UTC; checked on every request |
| `ipAddress` | TEXT | NULL                                   | Client IP at session creation          |
| `userAgent` | TEXT | NULL                                   | Client User-Agent at session creation  |
| `createdAt` | TEXT | NOT NULL                               | ISO 8601 UTC timestamp                 |
| `updatedAt` | TEXT | NOT NULL                               | ISO 8601 UTC timestamp                 |

### `account` Table

Links users to authentication providers. For email/password, `providerId = 'credential'` and `password` contains the hashed password.

| Column                  | Type | Constraints                            | Notes                                                          |
| ----------------------- | ---- | -------------------------------------- | -------------------------------------------------------------- |
| `id`                    | TEXT | PRIMARY KEY                            | UUID generated by better-auth                                  |
| `userId`                | TEXT | NOT NULL, FK → user(id) CASCADE DELETE |                                                                |
| `accountId`             | TEXT | NOT NULL                               | Provider-specific account identifier                           |
| `providerId`            | TEXT | NOT NULL                               | `'credential'` for email/password, `'google'` for Google OAuth |
| `accessToken`           | TEXT | NULL                                   | OAuth access token                                             |
| `refreshToken`          | TEXT | NULL                                   | OAuth refresh token                                            |
| `idToken`               | TEXT | NULL                                   | OAuth ID token                                                 |
| `accessTokenExpiresAt`  | TEXT | NULL                                   | ISO 8601 UTC                                                   |
| `refreshTokenExpiresAt` | TEXT | NULL                                   | ISO 8601 UTC                                                   |
| `scope`                 | TEXT | NULL                                   | OAuth scopes                                                   |
| `password`              | TEXT | NULL                                   | Argon2id hash for email/password provider                      |
| `createdAt`             | TEXT | NOT NULL                               | ISO 8601 UTC timestamp                                         |
| `updatedAt`             | TEXT | NOT NULL                               | ISO 8601 UTC timestamp                                         |

### `verification` Table

Used by better-auth for email verification tokens, password reset tokens, etc. Retained for future use even though email verification is disabled in this iteration.

| Column       | Type | Constraints | Notes                             |
| ------------ | ---- | ----------- | --------------------------------- |
| `id`         | TEXT | PRIMARY KEY |                                   |
| `identifier` | TEXT | NOT NULL    | Target identifier (email address) |
| `value`      | TEXT | NOT NULL    | The verification token            |
| `expiresAt`  | TEXT | NOT NULL    | ISO 8601 UTC                      |
| `createdAt`  | TEXT | NULL        | ISO 8601 UTC timestamp            |
| `updatedAt`  | TEXT | NULL        | ISO 8601 UTC timestamp            |

## Migration

Generated via better-auth CLI (`npx @better-auth/cli generate`) and stored as a D1 migration file.

**File**: `migrations/0001_better_auth_schema.sql`

```sql
CREATE TABLE `user` (
  `id` text NOT NULL,
  `name` text NOT NULL,
  `email` text NOT NULL UNIQUE,
  `emailVerified` integer NOT NULL DEFAULT 0,
  `image` text,
  `createdAt` text NOT NULL,
  `updatedAt` text NOT NULL,
  PRIMARY KEY(`id`)
);

CREATE TABLE `session` (
  `id` text NOT NULL,
  `userId` text NOT NULL REFERENCES `user`(`id`) ON DELETE CASCADE,
  `token` text NOT NULL UNIQUE,
  `expiresAt` text NOT NULL,
  `ipAddress` text,
  `userAgent` text,
  `createdAt` text NOT NULL,
  `updatedAt` text NOT NULL,
  PRIMARY KEY(`id`)
);

CREATE TABLE `account` (
  `id` text NOT NULL,
  `userId` text NOT NULL REFERENCES `user`(`id`) ON DELETE CASCADE,
  `accountId` text NOT NULL,
  `providerId` text NOT NULL,
  `accessToken` text,
  `refreshToken` text,
  `idToken` text,
  `accessTokenExpiresAt` text,
  `refreshTokenExpiresAt` text,
  `scope` text,
  `password` text,
  `createdAt` text NOT NULL,
  `updatedAt` text NOT NULL,
  PRIMARY KEY(`id`)
);

CREATE TABLE `verification` (
  `id` text NOT NULL,
  `identifier` text NOT NULL,
  `value` text NOT NULL,
  `expiresAt` text NOT NULL,
  `createdAt` text,
  `updatedAt` text,
  PRIMARY KEY(`id`)
);
```

## Runtime State Model

```
Application State
│
├── better-auth (manages)
│   ├── D1: user, session, account, verification tables
│   └── In-memory: rate limit counters (per isolate)
│
└── Worker (manages)
    └── Request scope: CSRF token (double-submit cookie)
```

**Note on rate limiting**: Better-auth uses in-memory rate limiting by default. This means rate limit counters reset on each cold start and are not shared across Workers isolates. This is acceptable for this iteration. If stronger guarantees are needed, a KV-backed rate limiter can be added in a future iteration.

## Session Lifecycle

```
Registration
  → POST /auth/register
  → better-auth creates: user row, account row
  → Redirect to /auth/login

Login
  → POST /auth/login
  → better-auth: validates credentials, creates session row
  → Response: Set-Cookie: better-auth.session={token}
  → Redirect to /?redirectTo or /

Request (authenticated)
  → GET /app/*
  → requireAuth middleware: auth.api.getSession({ headers })
  → better-auth: SELECT * FROM session WHERE token = ? AND expiresAt > ?
  → User context set on request, proceed

Logout
  → POST /auth/logout
  → better-auth: DELETE FROM session WHERE token = ?
  → Response: Clear-Cookie: better-auth.session
  → Redirect to /auth/login

Session expiry (passive)
  → auth.api.getSession() returns null when expiresAt < NOW
  → requireAuth middleware redirects to /auth/login
```

## CSRF State (Custom, for HTML Form Handlers)

The double-submit CSRF pattern stores state in two places simultaneously:

1. **Cookie**: `_csrf={token}; HttpOnly; Secure; SameSite=Strict; Path=/auth`
2. **Hidden form field**: `<input type="hidden" name="_csrf" value="{token}">`

The CSRF token is a 32-byte cryptographically random hex string generated using `crypto.getRandomValues()` (Web Standard API). The cookie is set when the form page is served (`GET /auth/login`, `GET /auth/register`) and validated on form submission (`POST /auth/login`, `POST /auth/register`, `POST /auth/logout`).
