# Octalysis Analysis Methodology

## Three Levels of Analysis

### Level 1: Core Drive Analysis
Identify which core drives and gamification techniques are present in the system.

**Steps:**
1. List all observable game mechanics and features
2. Map each mechanic to its corresponding core drive(s)
3. Identify which gamification techniques are being used
4. Note the intensity of each core drive (weak/moderate/strong)
5. Visualize on an octagon diagram (if applicable)

**Output:**
- List of active core drives
- Strength assessment for each drive
- Identified gamification techniques
- Initial observations about balance

---

### Level 2: Player Lifecycle Analysis
Analyze the system through different stages of the player journey.

**The Four Phases:**

#### 1. Discovery Phase
- How do users first learn about the system?
- What initial hooks attract attention?
- Which core drives are emphasized in marketing/first exposure?

**Key Drives to Consider:**
- Epic Meaning & Calling (Why should they care?)
- Social Influence (What are others saying?)
- Unpredictability (What mysteries await?)

#### 2. Onboarding Phase
- How are new users introduced to the system?
- What are the first actions users take?
- How quickly do users experience value?

**Key Drives to Consider:**
- Development & Accomplishment (Quick wins)
- Empowerment of Creativity (Early agency)
- Epic Meaning & Calling (Reinforce purpose)
- Beginner's Luck techniques

#### 3. Scaffolding Phase
- How does the system maintain engagement during regular use?
- What drives continued progression?
- How is complexity introduced?

**Key Drives to Consider:**
- All core drives should be balanced
- Development & Accomplishment (Clear progress)
- Ownership & Possession (Investment)
- Social Influence (Community building)

#### 4. Endgame Phase
- What keeps veteran users engaged?
- How does the system provide ongoing value?
- What prevents users from leaving?

**Key Drives to Consider:**
- Unpredictability (New surprises)
- Social Influence (Status, mentorship)
- Loss & Avoidance (Sunk cost)
- Development & Accomplishment (Mastery)

**Analysis Process:**
1. For each phase, conduct a Level 1 analysis
2. Identify which core drives are present and their strength
3. Look for gaps or overdependence on specific drives
4. Note transitions between phases
5. Assess if drives are appropriate for each phase

---

### Level 3: Player Type Analysis
Analyze how different player types experience the system across the lifecycle.

**Common Player Type Frameworks:**

#### Bartle's Four Player Types:
1. **Achievers** - Want to accomplish goals and complete tasks
   - Most motivated by: Development & Accomplishment, Ownership & Possession
   
2. **Explorers** - Want to discover and understand
   - Most motivated by: Unpredictability & Curiosity, Empowerment of Creativity
   
3. **Socializers** - Want to interact with others
   - Most motivated by: Social Influence & Relatedness
   
4. **Killers** - Want to compete and impose on others
   - Most motivated by: Development & Accomplishment, Social Influence (competitive aspects)

**Analysis Matrix:**
Create a matrix of Player Types × Lifecycle Phases

For each cell, analyze:
1. Which core drives are active?
2. How well does the system serve this player type at this phase?
3. What gaps exist?
4. What conflicts arise between player types?

**Process:**
1. Identify relevant player types (use Bartle's or custom types)
2. For each player type, conduct Level 2 analysis
3. Map how each type experiences each lifecycle phase
4. Identify where system serves or fails each type
5. Look for conflicts between player type needs
6. Note if system favors certain types over others

## Analysis Best Practices

### Balance Assessment
- **White Hat vs Black Hat:** Too much black hat creates stress; too much white hat can lack urgency
- **Left Brain vs Right Brain:** Too analytical can be boring; too creative can be overwhelming
- **Phase Appropriateness:** Different phases need different drive emphasis

### Common Patterns to Identify

**Positive Patterns:**
- Strong white hat in onboarding
- Balanced drive distribution in scaffolding
- Multiple paths for different player types
- Clear progression systems
- Social elements that don't feel forced

**Warning Signs:**
- Over-reliance on black hat drives
- Single dominant drive (except in onboarding)
- Weak endgame engagement
- Poor transitions between phases
- Ignoring entire player types
- Loss & Avoidance as primary retention mechanism

### Documentation Format

**For Each Level:**
1. Summary of findings
2. Visual representation (octagon diagram)
3. Specific observations by core drive
4. Identified strengths
5. Identified weaknesses
6. Recommendations for improvement

**Progressive Detail:**
- Level 1: Basic overview
- Level 2: Phase-by-phase breakdown
- Level 3: Detailed matrix with player types
