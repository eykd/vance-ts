## Architecture (Clean Architecture, 4 Layers)

```
src/
  domain/          Pure business logic, zero external deps
    entities/        AuthUser, AuthSession
    value-objects/   CommonPasswords (OWASP blocklist)
    services/        Argon2id password hasher (@noble/hashes)
    galaxy/          Procedural galaxy generation pipeline

  application/     Use-case orchestration
    ports/           AuthService, RateLimiter (interfaces only)
    use-cases/       SignInUseCase, SignUpUseCase, SignOutUseCase

  infrastructure/  Adapters implementing ports
    BetterAuthService   Maps better-auth responses to Result types
    DurableObjectRateLimiter  DO-backed atomic rate limiting
    auth.ts             better-auth singleton factory (D1 + Drizzle)

  presentation/    HTTP layer
    handlers/        ApiHandlers, AuthPageHandlers, StaticAssetHandler, AppPartialHandlers
    middleware/      requireAuth (session + CSRF)
    templates/       authLayout, login page, register page
    utils/           securityHeaders, cookieBuilder, extractClientIp,
                     redirectValidator, html (tagged-template escaping)
    generated/       assetPaths.ts (synced from Hugo build)

  di/              Composition root
    serviceFactory.ts  Lazy singleton wiring per isolate

  shared/          Cross-cutting utilities
    env.ts           Env interface (Cloudflare bindings)
    hex.ts           Binary-to-hex conversion

  worker.ts        Hono app entry point
  index.ts         Re-export
```

**Dependency rule enforced:** Domain <- Application <- Infrastructure/Presentation.
ESLint architecture boundary rules ready to uncomment in `eslint.config.mjs`.

## Domain Entities & Value Objects

### AuthUser

Represents an authenticated user in the domain layer.

| Field           | Type      | Notes                  |
| --------------- | --------- | ---------------------- |
| `id`            | `string`  | Unique identifier      |
| `email`         | `string`  | User's email           |
| `name`          | `string`  | Display name           |
| `emailVerified` | `boolean` | Verification status    |
| `createdAt`     | `string`  | ISO 8601 UTC timestamp |

### AuthSession

Represents an active user session.

| Field       | Type     | Notes                                  |
| ----------- | -------- | -------------------------------------- |
| `id`        | `string` | Unique identifier                      |
| `token`     | `string` | Cookie token, used for CSRF derivation |
| `userId`    | `string` | FK to AuthUser                         |
| `expiresAt` | `string` | Session expiration (ISO 8601 UTC)      |
| `createdAt` | `string` | Session creation (ISO 8601 UTC)        |

All fields are `readonly`.

### CommonPasswords (Value Object)

`ReadonlySet<string>` of 1000+ weak passwords from OWASP SecLists. Covers:

- OWASP top-100 passwords
- Numeric sequences and repeating digits
- Keyboard walk patterns (`asdfghjkl`, `qazwsx`, etc.)
- Common words, names, and word+number combos
- Long passwords (12+ chars): `passwordpassword1`, `thequickbrownfox`, etc.

Consumers must lowercase input before checking membership.

### Password Hasher (Domain Service)

**Algorithm:** Argon2id via `@noble/hashes` (pure JS, no WASM)

| Parameter   | Value                                    |
| ----------- | ---------------------------------------- |
| Memory      | 19,456 KiB (19 MiB) — OWASP 2023 minimum |
| Time cost   | 2 iterations                             |
| Parallelism | 1                                        |
| Key length  | 32 bytes (256-bit)                       |
| Salt        | 16 bytes (fresh per hash)                |

**Output format:** `argon2id$<memory>$<time>$<parallelism>$<salt-hex>$<derived-hex>` (self-describing)

**Verification:** Parses stored parameters, recomputes hash, uses constant-time comparison to prevent timing oracles. Returns `false` for any malformed input.

## Galaxy Generation Pipeline

Procedural content generation system in `src/domain/galaxy/`. All code is portable (no Node.js or Workers-specific APIs) and deterministic from a seed.

### Type System (`types.ts`)

**Star System Classification** (enum):

| Value            | Meaning                  |
| ---------------- | ------------------------ |
| `OIKUMENE`       | Established civilization |
| `UNINHABITED`    | No permanent population  |
| `LOST_COLONY`    | Abandoned settlement     |
| `HIDDEN_ENCLAVE` | Isolated community       |

**StarSystem** — complete star system record:

| Field            | Type               | Purpose                                                               |
| ---------------- | ------------------ | --------------------------------------------------------------------- |
| `id`, `name`     | `string`           | Identity                                                              |
| `x`, `y`         | `number`           | Coordinates                                                           |
| `isOikumene`     | `boolean`          | Part of established civilization                                      |
| `classification` | `Classification`   | Political/narrative category                                          |
| `density`        | `DensityData`      | neighborCount, environmentPenalty                                     |
| `attributes`     | `TerRating`        | technology, environment, resources                                    |
| `planetary`      | `PlanetaryData`    | size, atmosphere, temperature, hydrography                            |
| `civilization`   | `CivilizationData` | population, starport, government, factions, lawLevel                  |
| `tradeCodes`     | `string[]`         | Trade classification codes                                            |
| `economics`      | `EconomicsData`    | techLevel, perCapitaIncome, GWP, resourceMultiplier, worldTradeNumber |

**Route** — pre-computed navigable path between systems:

- `originId`, `destinationId`, `cost`, `path` (ordered Coordinate[])

**Configuration interfaces:** `GalaxyConfig` (spiral arm generation: center, size, turn, deg, arms, seed), `CostMapConfig` (grid parameters, noise weights), `PerlinConfig` (base + wall noise layers), `CaConfig` (cellular automata: fill probability, iterations, rule), `OikumeneConfig` (core exclusion, cluster radius, target count, radiate radius), `RouteConfig` (max range), `CostMapQuantization` (byte-to-cost mapping).

**GenerationStats** — pipeline output metrics: totalSystems, oikumeneSystems, beyondSystems (uninhabited/lostColonies/hiddenEnclaves), oikumeneRoutes, averageRouteCost.

### PRNG (`prng.ts`)

**Mulberry32** — seedable 32-bit PRNG (~20 lines, zero dependencies).

- `random(): number` → uniform float [0, 1)
- `randint(min, max): number` → uniform integer [min, max] inclusive
- Deterministic and identical across Node.js and Cloudflare Workers.

### Dice (`dice.ts`)

- **`roll4dF(rng)`** — Roll 4 Fate dice (4dF), range [-4, +4]. Each die: {-1, 0, +1}.
- **`rollNdS(rng, count, sides)`** — Roll N dice with S sides (NdS), range [count, count×sides].

### Spatial Hash (`spatial-hash.ts`)

Grid-based spatial indexing for efficient neighbor lookups.

| Method                      | Complexity | Purpose                          |
| --------------------------- | ---------- | -------------------------------- |
| `insert(index, x, y)`       | O(1)       | Register a system at coordinates |
| `queryRadius(x, y, radius)` | O(k)       | Find all systems within radius   |

Reduces full-galaxy neighbor counting from O(n²) to O(n). Uses Chebyshev box pre-filter → Euclidean distance refinement.

### A\* Pathfinding (`pathfinding.ts`)

Grid-based A\* with 8-directional movement over a cost map.

**Key components:**

- **BinaryHeap** — min-heap priority queue (parallel arrays for nodes + priorities), O(log n) push/pop
- **Octile distance heuristic** — admissible for 8-directional grids: `minCost × (dx + dy + (√2 - 2) × min(dx, dy))`
- **Cost quantization** — Uint8Array bytes (0–255) mapped to [minCost, maxCost] float range
- **Movement costs** — cardinal: cellCost × 1, diagonal: cellCost × √2

**`createPathfinder(costData, width, height, quantization, config?)`** returns a `Pathfinder` with:

- `findPath(start, end): PathfindingResult | null`

**Performance optimizations:**

- Typed arrays (Float64Array, Uint8Array, Int32Array) for cache efficiency
- Early termination on max cost threshold
- Path reconstruction via Int32Array backpointer chain

### How It Fits Together

```
Seed → Mulberry32 PRNG
         ↓
  Spiral arm placement → star coordinates
         ↓
  SpatialHash → neighbor density for each system
         ↓
  Dice rolls (4dF, NdS) → planetary, civilization, economics attributes
         ↓
  Classification (Oikumene selection via config)
         ↓
  Cost map (Perlin noise + cellular automata) → terrain grid
         ↓
  A* pathfinding → inter-system routes with travel costs
         ↓
  StarSystem[] + Route[] + GenerationStats
```

## Security Posture

- **CSRF:** Double-submit token (form field + `__Secure-csrf` cookie), session-bound via HMAC-SHA256
- **Rate limiting:** IP-based via Durable Object (RateLimitDO), 5 attempts per window (15 min sign-in, 5 min register)
- **Timing oracle defense:** PBKDF2 dummy hash on every non-rate-limited failure
- **Email enumeration prevention:** Same response for email_taken and success on sign-up
- **Password validation:** 12-128 chars (better-auth) + OWASP common-password blocklist
- **Sessions:** `__Host-` prefixed cookies, HttpOnly, Secure, SameSite, 30-day TTL
- **Headers:** HSTS (2yr, preload), CSP, X-Frame-Options DENY, Referrer-Policy, Permissions-Policy
- **HTML escaping:** Tagged template literal with auto-escape (`html` utility)
- **Redirect validation:** Blocks `//`, `/api/`, `/auth/` paths; localhost-base URL parsing
- **IP extraction:** CF-Connecting-IP only (ignores X-Forwarded-For to prevent spoofing)

Automated loop: queries `br ready`, invokes `/sp:next`, polls for completion, repeats.

10 retries with exponential backoff. Lock file prevents concurrent runs.
